A browser opens, you click through the real flow, then say done — or just close the window. Files are written automatically.

What you get

Every recording writes three files. The one you read is the Markdown doc.

📄

.md flow doc

A readable write-up: overview, a Mermaid sequence diagram, an ordered call table, and per-call detail.

🤖

.flow.json

Structured, ordered data with request/response bodies — for Claude/Cursor to build even richer docs.

🔧

.har

Standard HAR 1.2 — open it in Chrome DevTools, Postman, Insomnia or Charles.

Install in 10 seconds

Nothing to download. Add this to your Claude or Cursor MCP config — npx runs it from npm and uses the browsers you already have.

{
  "mcpServers": {
    "browser-flow-tracker": {
      "command": "npx",
      "args": ["-y", "browser-flow-tracker@latest"]
    }
  }
}

Claude Code shortcut: claude mcp add browser-flow-tracker -- npx -y browser-flow-tracker@latest

  1. Add the config above and restart your AI app.
  2. Say the phrase: “let's record the session for this url …”. A real browser window opens.
  3. You navigate the flow — log in if needed (remembered next time).
  4. Say “done” or close the browser. The doc + JSON + HAR are written for you.

Why it's better than the console

It speaks the browser's own DevTools Protocol, then does the tedious parts for you.

🧹

Smart-filtered

Drops images, fonts, CSS and analytics/telemetry (PostHog, GA4, RUM) so you see only the real API calls.

🔐

Safe to share

Auth headers, cookies and tokens are redacted by default. Log in once — the private profile remembers you.

🧭

You drive

The capture is user-driven: it records your real, authenticated flow instead of guessing. It never clicks for you.

🌐

Your browsers

Works with Brave, Chrome, Arc, Edge and other Chromium browsers. Auto-picks a free debug port — no setup.
